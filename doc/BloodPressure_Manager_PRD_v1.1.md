## 1. 제품 개요

### 1.1 배경 및 목적

BloodPressure Manager는 고혈압 자가 관리를 위한 모바일 최적화 웹앱입니다.  
v1.1에서는 의학계의 공통 인식인 **"혈압은 나이에 따라 자연스럽게 상승하며, 연령대별로 다른 판정 기준이 필요하다"** 는 원칙을 앱 전반에 반영했습니다.

**v1.1 핵심 목표:**

- 연령대(6구간)별 차등 혈압 판정 기준 적용
- 3회 측정 → 평균값을 1회 데이터로 통합 관리 (v1.0 유지)
- 임의 기간 지정 혈압 추이 PDF 리포트 생성 (v1.0 유지)
- 하네스 기법 기반 입력 검증 + AI 판정 시스템 (v1.0 유지 + 연령 매트릭스 확장)

### 1.2 핵심 사용자

| 사용자 유형 | 설명 | 주요 니즈 |
|------------|------|-----------|
| 일반 환자 | 고혈압 자가 관리자 | 간편한 기록, 추세 파악 |
| 의사 제출용 사용자 | 진료 데이터 준비 필요자 | PDF 리포트 생성, 기간 지정 |
| 고령 사용자 | 디지털 기기 비숙련자 | 단순 UI, 큰 글씨, 직관적 흐름 |

### 1.3 연령대 기준 도입 배경 — 의학적 근거

> 📌 **국제 가이드라인 3종 참조: AHA/ACC 2025 · ESH 2023 · ESC 2024**

| 기관 | 고혈압 진단 기준 | 연령 차등 여부 |
|------|----------------|--------------|
| AHA/ACC 2025 (미국) | SYS ≥ 130 또는 DIA ≥ 80 | 단일 기준 / 단, 65세↑ 완화 권고 병기 |
| ESH 2023 (유럽) | SYS ≥ 140 또는 DIA ≥ 90 | **연령별 치료 목표 차등 명시** |
| ESC 2024 (유럽) | SYS ≥ 140 또는 DIA ≥ 90 | 허약·고령·기립성 저혈압 시 완화 기준 |
| 한국 질병관리청 | SYS ≥ 140 또는 DIA ≥ 90 | AHA 기준 준용, 65세↑ 별도 관리 권고 |

### 1.4 제품 비전

> **"측정 → 기록 → 분석 → 보고" 전 과정을 자동화하여,  
> 환자와 의료진 모두가 신뢰할 수 있는 혈압 관리 플랫폼**

## 2. 연령대별 혈압 판정 기준 (v1.1 핵심 신규)

### 2.1 연령대 구분 및 기준값 정의

> ⚕️ **아래 기준은 AHA 2025 · ESH 2023 · ESC 2024 · 한국 질병관리청 데이터를 종합한 앱 전용 참고 기준이며,  
> 최종 진단은 반드시 의료 전문가에게 받아야 합니다.**

| 연령대 | 평균 혈압 | 🟢 정상 | 🟡 주의 | 🔴 고혈압 의심 | 맥박 | 임상 비고 |
|--------|---------|---------|---------|--------------|------|-----------|
| **20대** | 115/75 | < 120/80 | 120~124 / 80~83 | ≥ 125/84 | 60~90 | 혈관 탄성 최고 → 기준 가장 엄격 |
| **30대** | 118/77 | < 122/80 | 122~129 / 80~84 | ≥ 130/85 | 60~90 | 생활습관 관리 시작 시기 |
| **40대** | 122/80 | < 125/82 | 125~134 / 82~86 | ≥ 135/87 | 60~90 | 대사증후군 위험 증가 |
| **50대** | 128/82 | < 130/84 | 130~139 / 84~88 | ≥ 140/89 | 60~90 | 갱년기·동맥경화 영향 |
| **60대** | 134/83 | < 135/85 | 135~144 / 85~89 | ≥ 145/90 | 55~90 | ESH 기준 차등 적용 구간 |
| **70대+** | 140/82 | < 140/85 | 140~149 / 85~89 | ≥ 150/90 | 55~90 | 기립성 저혈압 위험 → 완화 기준 |

### 2.2 연령대 기준 핵심 원칙

| # | 원칙 | 상세 설명 |
|---|------|-----------|
| ① | **혈압은 나이와 함께 상승** | 혈관 탄성 저하·동맥경화로 나이 들수록 수축기 혈압이 자연 상승하며, 이를 무조건 고혈압으로 판정하면 임상적으로 부정확 |
| ② | **노년층 완화 기준 적용** | 70대 이상에서 120/80 강제 유지 시 기립성 저혈압 → 낙상 위험. ESH 2023·ESC 2024 모두 고령자 완화 목표 명시 |
| ③ | **동일값 연령 보정 예시** | SYS 138 / DIA 84 → 20대: "주의", 60대: "정상" (동일 수치도 연령대 따라 판정이 달라짐) |
| ④ | **앱 면책 조항 강제 출력** | 모든 판정 결과 하단에 "이 결과는 참고용이며 의료 진단이 아닙니다" 문구 의무 표시 |

## 3. 핵심 기능 정의 (v1.1 업데이트)

### 3.1 연령대 입력 — 신규 필수 필드

v1.1부터 연령대 입력은 선택이 아닌 **필수 항목**입니다. 연령대 미선택 시 측정값 저장이 불가합니다.

| 항목 | 입력 방식 | 비고 |
|------|-----------|------|
| **연령대** | 토글 버튼 6개 (20대/30대/40대/50대/60대/70대+) | 최초 1회 설정 후 앱 설정에서 변경 가능 (매번 입력 불필요) |
| 저장 위치 | 앱 설정(프로필) + 측정 세션 | 프로필에 고정, 측정 시 자동 적용 / 개별 변경 가능 |
| 검증 | 미선택 시 저장 버튼 비활성화 | 하네스 ① 입력 검증에 포함 |

### 3.2 3회 측정 → 1회 데이터 통합 (v1.0 유지 + 연령 판정 연동)

3회 측정 후 평균값을 산출하고, 해당 연령대 기준값에 따라 최종 판정을 수행합니다.

| 측정 | SYS | DIA | PUL | 상태 |
|------|-----|-----|-----|------|
| 1회차 | 128 | 82 | 74 | 입력 완료 |
| 2회차 | 124 | 79 | 72 | 입력 완료 |
| 3회차 | 126 | 80 | 73 | 입력 완료 |
| **평균 (저장)** | **126** | **80** | **73** | **정상 (50대 기준)** |

> ⚠️ 동일한 126/80이라도 **20대 기준 적용 시: "주의"** / **50대 기준 적용 시: "정상"**

**상세 로직:**

- 3회 측정 완료 전: 임시 저장 상태, 저장 버튼 비활성화
- 3회 완료 시: 자동 평균 계산 → 미리보기 표시 → 저장 버튼 활성화
- 이상값 탐지: 회차 간 수축기 차이 20mmHg 이상 시 경고 표시 및 재측정 권고
- 저장 데이터: 3회 개별 측정값 + 평균값 + 메타데이터(시간대, 팔, 자세) 모두 보관

### 3.3 PDF 리포트 (v1.0 유지 + 연령대 정보 추가)

- PDF 표지에 측정자 연령대 및 적용된 판정 기준 명시
- 통계 요약 카드에 "연령대 기준 대비 분석" 섹션 추가
- 데이터 테이블에 연령대 보정 판정 컬럼 추가
- 면책 조항: "본 리포트의 판정 기준은 [연령대] 기준이며 참고용입니다" 문구 고정

**PDF 리포트 구성 요소:**

| 구성 요소 | 내용 |
|-----------|------|
| 표지 | 환자 이름, 연령대, 측정 기간, 생성 날짜 |
| 통계 요약 | 기간 내 평균/최고/최저 수축기·이완기·맥박, 측정 횟수 |
| 혈압 추이 차트 | 꺾은선 그래프 (수축기/이완기/정상 범위 기준선) |
| 측정 데이터 테이블 | 날짜, 시간대, 팔, 자세, SYS/DIA/PUL, 연령 보정 판정, 메모 |
| 의료진 메모란 | 의사가 직접 기입할 수 있는 빈 서명/메모 공간 |
| 면책 조항 | 고정 출력, 저작권 포함 |

## 4. 하네스(Harness) 아키텍처 — 연령 매트릭스 확장

### 4.1 3중 하네스 구조 (v1.1 업데이트)

```
하네스 ①  입력 검증 (연령대 필수 포함)
    ↓
하네스 ②  연령대별 AI 판정 에이전트
    ↓
하네스 ③  출력 형식 + 면책 문구 검증
```

### 4.2 하네스 ① — 입력 검증 기준표

| 항목 | 허용 최솟값 | 허용 최댓값 | 오류 처리 |
|------|-----------|-----------|-----------|
| **연령대 (신규)** | 20대 | 70대+ | **미선택 시 저장 버튼 비활성화, 경고 표시** |
| 수축기 (SYS) | 60 mmHg | 250 mmHg | 범위 초과 시 입력 차단 + 인라인 경고 |
| 이완기 (DIA) | 40 mmHg | 150 mmHg | SYS < DIA 시 논리 오류 경고 |
| 맥박 (PUL) | 30 bpm | 200 bpm | 범위 초과 시 입력 차단 + 경고 |
| 회차 간 편차 | — | ±20 mmHg | 편차 초과 시 재측정 권고 팝업 |

### 4.3 하네스 ② — AI 판정 에이전트 프롬프트 (v1.1)

**입력 JSON:**

```json
{
  "age_group": "20대|30대|40대|50대|60대|70대+",
  "sys": 숫자,
  "dia": 숫자,
  "pul": 숫자,
  "time": "아침|저녁"
}
```

**판정 기준 (age_group에 따라 아래 기준값 적용):**

```
20대: 정상 <120/80 | 주의 120~124/80~83 | 고혈압의심 ≥125/84
30대: 정상 <122/80 | 주의 122~129/80~84 | 고혈압의심 ≥130/85
40대: 정상 <125/82 | 주의 125~134/82~86 | 고혈압의심 ≥135/87
50대: 정상 <130/84 | 주의 130~139/84~88 | 고혈압의심 ≥140/89
60대: 정상 <135/85 | 주의 135~144/85~89 | 고혈압의심 ≥145/90
70대+: 정상 <140/85 | 주의 140~149/85~89 | 고혈압의심 ≥150/90
```

**출력 (JSON만 반환):**

```json
{
  "status": "정상|주의|고혈압 의심",
  "age_adjusted": true,
  "message": "20자 이내",
  "advice": "생활습관 조언 1가지",
  "disclaimer": "이 결과는 참고용이며 의료 진단이 아닙니다."
}
```

### 4.4 하네스 ③ — 출력 검증 항목

- `status` 필드 존재 및 허용값 확인 (v1.0 유지)
- `age_adjusted: true` 포함 여부 확인 (**신규** — 연령 보정 적용 여부)
- `disclaimer` 필드 존재 및 면책 문구 포함 확인 (**신규**)
- `message` 20자 이하 (v1.0 유지)
- 의학적 확정 진단 금칙어 필터 (v1.0 유지)

| 계층 | 기술 | 선택 이유 |

## 7. 비기능 요구사항

### 7.1 성능

- 페이지 초기 로딩: **3초 이내** (3G 환경 기준)
- PDF 생성 완료: **5초 이내** (30일치 데이터 기준)
- AI 판정 응답: **3초 이내**

### 7.2 보안 및 개인정보

- 모든 혈압 데이터는 기기 로컬(IndexedDB)에만 저장 — **서버 전송 없음**
- 연령대 정보 포함 AI 판정 API 호출 시 개인 식별 정보 미포함
- PDF 파일 내 개인정보는 사용자가 직접 입력

### 7.3 접근성

- WCAG 2.1 AA 수준 준수
- 고령 사용자(70대+) 대상 폰트 최소 **18px**, 터치 영역 **48px** 이상
- 색맹 사용자를 위한 아이콘+색상 병행 표시  
  - 정상: ✅ 초록  
  - 주의: ⚠️ 노랑  
  - 고혈압 의심: 🔴 빨강
