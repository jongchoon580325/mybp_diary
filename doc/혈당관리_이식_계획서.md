## 1. 이식 전략 개요

### 1.1 기본 방침

기존 혈압 관리 앱의 **모든 인프라를 그대로 재사용**하고, 혈당 관리 기능을 **병렬 도메인**으로 추가한다.
새로운 앱을 처음부터 만들지 않으며, 공유 가능한 코드(Auth, Firestore, 디자인 토큰, Modal/Toast, BottomNav 등)는 100% 재활용한다.

### 1.2 이식 가능 자산 목록

| 자산 | 재사용 여부 | 비고 |
|------|-----------|------|
| Firebase Auth (Google + 익명) | ✅ 100% 재사용 | 변경 없음 |
| Firebase Firestore 연동 구조 | ✅ 재사용 | 컬렉션 경로만 추가 |
| IBpRepository 패턴 | ✅ 구조 복제 | IGlucoseRepository 신규 작성 |
| useAuth / AuthContext | ✅ 100% 재사용 | 변경 없음 |
| useModal / ToastNotification | ✅ 100% 재사용 | 변경 없음 |
| Modal.tsx | ✅ 100% 재사용 | 변경 없음 |

| BottomNav | ⚠️ 탭 구조 변경 필요 | 혈압/혈당 전환 탭 추가 |

## 3. 신규 타입 정의 (src/types/index.ts 추가)

기존 혈압 타입은 변경하지 않고, 혈당 전용 타입을 **추가**한다.

```typescript
// ─── 혈당 측정 태그 ──────────────────────────────────────────────────────────
export type MealTag = '공복' | '식전' | '식후 1h' | '식후 2h' | '취침 전';

// ─── 당뇨 유형 ────────────────────────────────────────────────────────────────
export type DiabetesType = '1형' | '2형' | '임신성' | '없음';

// ─── 혈당 판정 상태 ──────────────────────────────────────────────────────────
export type GlucoseStatus = '정상' | '주의' | '고혈당' | '저혈당';

// ─── 혈당 기록 (Firestore/IndexedDB 저장 단위) ───────────────────────────────
export interface GlucoseRecord {
  record_id: string;          // UUID
  measured_at: string;        // ISO 8601 DateTime
  glucose_level: number;      // 혈당 수치 (mg/dL)
  meal_tag: MealTag;          // 측정 시점 태그
  insulin?: number;           // 인슐린 투여량 (단위), 선택
  carbs?: number;             // 탄수화물 섭취량 (g), 선택
  exercise?: string;          // 운동 종류 및 시간, 선택
  note?: string;              // 메모, 선택
  status: GlucoseStatus;      // 판정 결과
  created_at: string;         // ISO 8601
}

// ─── 혈당 목표 범위 (사용자 설정) ────────────────────────────────────────────
export interface GlucoseTarget {
  target_min: number;   // 목표 최솟값 (mg/dL), 기본 70
  target_max: number;   // 목표 최댓값 (mg/dL), 기본 140
  diabetes_type: DiabetesType;
}

// ─── 혈당 검증 오류 ──────────────────────────────────────────────────────────
export interface GlucoseValidationError {
  field: 'glucose_level' | 'meal_tag';
  message: string;
}
```

## 4. Firestore 데이터 구조 추가

기존 혈압 컬렉션(`users/{uid}/sessions`)은 유지하고, 혈당 전용 컬렉션을 추가한다.

```
users/{uid}/
  sessions/{session_id}     ← 기존 혈압 (변경 없음)
  glucose/{record_id}       ← 신규 혈당
  profile                   ← 기존 설정 + 혈당 목표 필드 추가
```

### 4.1 Firestore 보안 규칙 추가

```javascript
// 기존 rules에 glucose 컬렉션 추가
match /users/{uid}/glucose/{recordId} {
  allow read, write: if request.auth != null && request.auth.uid == uid;
}
```

### 4.2 glucose 문서 구조

```typescript
// Firestore 저장 시 추가 필드 (GlucoseRecord + 아래)
{
  ...GlucoseRecord,
  uid: string,              // 보안 규칙 검증용
  created_at_ts: Timestamp  // 서버 타임스탬프 (쿼리용)
}
```

```typescript
// useDB.ts 패턴 복제
export function useGlucoseDB(): IGlucoseRepository {
  const { user } = useAuthContext();
  return useMemo(() => {
    if (user && !user.isAnonymous) return createFirestoreGlucoseRepository(user.uid);
    return createIndexedDBGlucoseRepository();
  }, [user?.uid, user?.isAnonymous]);
}
```

## 11. IndexedDB 스키마 확장

기존 `bp-manager-db`에 `glucose` 스토어 추가 (DB 버전 2로 업그레이드).

```typescript
// src/services/db.ts 수정
const DB_VERSION = 2;  // 1 → 2

upgrade(db, oldVersion) {
  // 기존 sessions 스토어 유지 (oldVersion === 1이면 스킵)
  if (oldVersion < 2) {
    const store = db.createObjectStore('glucose', { keyPath: 'record_id' });
    store.createIndex('by-date',     'measured_at',   { unique: false });
    store.createIndex('by-meal-tag', 'meal_tag',      { unique: false });
    store.createIndex('by-status',   'status',        { unique: false });
  }
}
```
