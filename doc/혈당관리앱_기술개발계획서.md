## 1. 프로젝트 개요

### 1.1 목적

본 프로젝트는 개인 사용자가 혈당 수치를 체계적으로 기록·관리·분석할 수 있는 웹 기반 애플리케이션을 개발하는 것을 목표로 한다. 상용 앱의 복잡한 구독 모델이나 불필요한 기능 없이, 실사용 중심의 직관적인 혈당 관리 도구를 제공한다.

### 1.2 핵심 기능 요약

| 우선순위 | 기능 | 설명 |
|----------|------|------|
| 1 | 데이터 입력 및 기록 | 혈당 수치, 측정 시간, 식사/운동/약물 정보 |
| 2 | 시각화 및 트렌드 분석 | 일간/주간/월간 그래프, 통계 요약 |
| 3 | 데이터 저장 및 백업 | Firebase Firestore 기반 실시간 동기화 |
| 4 | 알림 및 리마인더 | 측정 시간 알림, 목표 범위 이탈 경고 |
| 5 | 식사·운동·약물 연동 메모 | 혈당 상관관계 분석용 부가 정보 기록 |
| 6 | PDF 출력 기능 | 의료 상담용 보고서 출력 |

## 5. Firestore 데이터 구조

```
users/
  {userId}/
    profile/
      - name: string
      - targetMin: number        // 목표 최저 혈당
      - targetMax: number        // 목표 최고 혈당
      - diabetesType: string     // 타입1 / 타입2 / 임신성 / 없음

    records/
      {recordId}/
        - glucoseLevel: number   // 혈당 수치 (mg/dL)
        - measuredAt: timestamp  // 측정 시간
        - mealTag: string        // 공복 / 식전 / 식후 1h / 식후 2h
        - insulin: number        // 인슐린 투여량 (단위)
        - carbs: number          // 탄수화물 섭취량 (g)
        - exercise: string       // 운동 종류 및 시간
        - note: string           // 메모
        - createdAt: timestamp
```

## 6. PDF 출력 기능 상세

### 6.1 구현 방식

`jsPDF` + `html2canvas` 조합을 사용하여 브라우저에서 직접 PDF를 생성한다. 서버 없이 클라이언트 사이드에서 처리하므로 Firebase Functions 불필요.

### 6.2 출력 내용

- **기간 요약:** 선택 기간(일/주/월)의 평균, 최고, 최저 혈당
- **혈당 추이 그래프:** Recharts 차트 → Canvas → PDF 삽입
- **상세 기록 테이블:** 날짜, 수치, 식사 태그, 메모
- **통계 분석:** 목표 범위 내 비율(TIR), 고혈당/저혈당 발생 횟수

### 6.3 출력 예시 구조 (A4 기준)

```
┌────────────────────────────────────┐
│ 혈당 관리 보고서                    │
│ 기간: 2026.03.01 ~ 2026.03.30      │
├────────────────────────────────────┤
│ [요약 통계]                         │
│  평균: 118 mg/dL   TIR: 78%        │
├────────────────────────────────────┤
│ [혈당 트렌드 그래프]                 │
│  ▓▓▓▓▓▒▒▒▒▒▓▓▓▓▓ (30일 추이)      │
├────────────────────────────────────┤
│ [상세 기록 테이블]                   │
│  날짜 | 수치 | 태그 | 메모          │
└────────────────────────────────────┘
```

## 9. 보안 고려사항

- **Firebase 보안 규칙:** 인증된 사용자만 자신의 데이터에 접근 가능하도록 Firestore Rules 설정
- **환경변수 관리:** Firebase API 키는 `.env.local`에 저장, Git에 커밋하지 않음
- **개인 데이터 보호:** 의료 데이터 특성상 외부 공유 기능 미구현 (개인 전용)

```javascript
// firestore.rules 예시
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null 
                         && request.auth.uid == userId;
    }
  }
}
```
